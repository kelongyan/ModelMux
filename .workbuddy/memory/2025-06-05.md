## 2025-06-05 — 前端 UI 布局优化

### 问题诊断
全面 review 了 10 个 CSS 文件和 5 个页面组件，锁定 7 个布局不协调点：

1. **Dashboard 左右比例倒置** — `0.9fr : 1.6fr` 让活跃 Provider 面板（核心信息）只占 36%
2. **Provider 卡片过度拉伸** — `auto-fit` + `1fr` 让少数 Provider 时卡片撑满整行
3. **内容区内边距堆叠过度** — 从视口到实际内容高达 58px 的 padding
4. **设置页表单 1/4 化** — 外层 2 列 + 内层 md=12 导致字段仅占 25% 宽
5. **事件栏列宽逼仄** — 时间和级别列刚够放内容
6. **间距尺度混乱** — 8/10/12/14/16/18/22/28/32/38px 无规则散布
7. **Stats KPI 卡片松垮** — 112px 高 + 24px padding 对数字卡片偏大
