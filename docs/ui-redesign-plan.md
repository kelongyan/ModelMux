# ModelMux 前端 UI 优化设计方案

## 现状诊断

### 整体评价

当前 UI 采用 "Anthropic Warm" 暖色调设计语言——以米色/奶油色为底、赤陶色 (terracotta) 为主色、衬线字体 (Times New Roman / Libre Baskerville) 做标题。整体氛围偏文学/出版风格，对于一个**技术运维管理面板**来说存在以下核心问题。

### 五大核心痛点

**1. 表面层次感不足 — "一片糊"**

5 级表面亮度从 `#ddd8ce` 到 `#f5f2ea`，明度跨度仅约 12%，卡片、背景、侧边栏之间几乎看不出边界。整个界面像是蒙了一层灰纱，信息区块缺乏视觉分离。

```
当前表面梯度（明度差太小）:
L1 #ddd8ce ████████████  → L5 #f5f2ea ████████████████
     仅 12% 明度跨度，肉眼难以区分
```

**2. 字体选择与场景不匹配**

- 标题使用 `"Times New Roman", Georgia, serif`——这是文学排版字体，不是 UI 字体
- 正文使用 `Libre Baskerville`——同样是衬线体，在小字号下可读性不如无衬线体
- 管理面板需要的是清晰、现代、高辨识度的 UI 字体

**3. 主色调沉闷**

赤陶色 `#b85f45` 偏暗偏土，作为唯一品牌色显得沉闷。用于按钮、链接、高亮时缺乏活力和引导感，尤其在暗色模式下更为明显。

**4. 信息密度不均**

- Dashboard 页信息太稀疏，KPI 不够突出
- Providers 页（773 行 "God Component"）信息太密集，操作按钮拥挤
- Stats 页图表和表格缺少明确的视觉节奏

**5. 交互反馈薄弱**

- 表格行 hover 效果不明显
- 卡片 hover lift 动画 (`translateY(-4px)`) 幅度太小
- 页面切换动画缺少方向感
- 加载态骨架屏覆盖不全

## 优化方案

### Phase 1: 视觉基础重构（改动最大，影响全局）

#### 1.1 配色体系升级

**目标**: 从 "文艺暖" 转向 "现代科技暖" —— 保留暖色基因，但增加对比度、活力和层次感。

**表面层重新设计**:

```
Light Mode — 拉大明度梯度，引入纯白卡片层
  Canvas:    #f0ece4  (暖灰底色)
  L1 Inset:  #e8e3da  (内嵌区域)
  L2 Card:   #ffffff  (纯白卡片 ← 关键变化)
  L3 Raised: #ffffff  (浮起卡片)
  L4 Active: #fef0e4  (选中高亮)

Dark Mode — 更深的底色 + 更亮的卡片
  Canvas:    #0f0f0e  (近黑)
  L1 Inset:  #1a1917  (内嵌)
  L2 Card:   #232220  (卡片)
  L3 Raised: #2d2c28  (浮起)
  L4 Active: rgba(56, 152, 236, 0.12)  (选中高亮)
```

**主色升级**:

```
当前: #b85f45 (赤陶 — 偏暗偏土)
方案: #2563eb (科技蓝 — 明亮、专业、有引导感)

备选方案 (如果你想保留暖色基因):
  方案A: #e07a3a (活力橙 — 保留暖调，增加活力)
  方案B: #6366f1 (靛蓝紫 — 现代、有科技感)
  方案C: #0ea5e9 (天蓝 — 清新、开放)
```

> 说明：主色变更是**最大的设计决策**，需要你拍板。蓝色最安全专业，橙色保留个性，紫色现代前卫。

**语义色微调**:
```
Success:  #22c55e → 更亮的绿（当前 #5aab6a 偏暗）
Warning:  #f59e0b → 更鲜明的琥珀（当前 #d4a017 偏暗）
Error:    #ef4444 → 更标准的红（当前 #c44848 偏暗）
```

#### 1.2 字体体系重做

**目标**: 全面切换无衬线字体栈，提升可读性和现代感。

```css
/* 标题 — 去掉衬线体 */
--font-display: "Inter", "SF Pro Display", "PingFang SC", "Microsoft YaHei UI", system-ui, sans-serif;

/* 正文 */
--font-body: "Inter", "SF Pro Text", "PingFang SC", "Microsoft YaHei UI", system-ui, sans-serif;

/* 等宽（保持不变） */
--font-mono: "Cascadia Code", "JetBrains Mono", "SF Mono", Menlo, Consolas, monospace;
```

**字号体系收紧** (当前用了太多不同字号，建议归一到 8 级):
```
--text-xs:   0.75rem   (12px — 辅助标签)
--text-sm:   0.8125rem (13px — 次要文本)
--text-base: 0.875rem  (14px — 正文)
--text-md:   0.9375rem (15px — 强调正文)
--text-lg:   1.0625rem (17px — 小标题)
--text-xl:   1.25rem   (20px — 区块标题)
--text-2xl:  1.5rem    (24px — 页面标题)
--text-3xl:  1.875rem  (30px — 大数字/KPI)
```

**字重规范**:
```
标题: 600 (semibold)
正文: 400 (regular)
标签: 500 (medium)
数字: 600 (semibold) + tabular-nums
```

#### 1.3 间距体系规范化

当前间距完全靠手写数值，没有系统。引入 4px 基准的间距阶梯:

```css
--space-1: 4px;
--space-2: 8px;
--space-3: 12px;
--space-4: 16px;
--space-5: 20px;
--space-6: 24px;
--space-8: 32px;
--space-10: 40px;
--space-12: 48px;

--radius-sm: 6px;
--radius-md: 10px;
--radius-lg: 14px;
--radius-xl: 20px;
--radius-pill: 999px;
```

#### 1.4 阴影体系

```css
/* Light */
--shadow-xs: 0 1px 2px rgba(0,0,0,0.04);
--shadow-sm: 0 1px 3px rgba(0,0,0,0.06), 0 1px 2px rgba(0,0,0,0.04);
--shadow-md: 0 4px 6px -1px rgba(0,0,0,0.07), 0 2px 4px rgba(0,0,0,0.04);
--shadow-lg: 0 10px 15px -3px rgba(0,0,0,0.08), 0 4px 6px rgba(0,0,0,0.04);

/* Dark — 用更明显的边框代替阴影 */
--shadow-xs: 0 0 0 1px var(--mm-border);
--shadow-sm: 0 0 0 1px var(--mm-border), 0 1px 3px rgba(0,0,0,0.3);
--shadow-md: 0 0 0 1px var(--mm-border), 0 4px 6px rgba(0,0,0,0.3);
--shadow-lg: 0 0 0 1px var(--mm-border), 0 10px 15px rgba(0,0,0,0.3);
```

### Phase 2: 布局与导航优化

#### 2.1 侧边栏重设计

**当前**: 250px 固定宽度，纯文字导航链接，背景与主区域几乎同色。

**优化方向**:

```
┌──────────┐  ┌──────────────────────────────────┐
│          │  │ Header: 面包屑 + 全局状态         │
│  LOGO    │  ├──────────────────────────────────┤
│          │  │                                  │
│  ┌─────┐ │  │                                  │
│  │ 总览 │ │  │                                  │
│  │ 提供商│ │  │       主内容区域                  │
│  │ 统计  │ │  │       (白色卡片背景)               │
│  │ 设置  │ │  │                                  │
│  │ 事件  │ │  │                                  │
│  │ 关于  │ │  │                                  │
│  └─────┘ │  │                                  │
│          │  │                                  │
│  ─────── │  │                                  │
│  主题切换 │  │                                  │
│  v1.2.3  │  │                                  │
└──────────┘  └──────────────────────────────────┘
```

- 宽度收窄至 **220px**（或可折叠到 64px 图标模式）
- 侧边栏底色用 **略深一级** 的表面色，与白色内容区形成明确对比
- 导航项增加 **左侧激活指示条** (3px solid primary)
- 激活项背景用主色 8% 透明度填充
- 图标从纯文字改为 **图标 + 文字** 组合
- 底部放置版本号 + 主题切换

#### 2.2 顶栏优化

- 左侧显示 **面包屑导航**（如 "提供商 > OpenAI"），替代当前简单的页面标题
- 右侧保留健康状态 + key 统计，增加一个 **快速搜索** 入口（Cmd+K 快捷键）
- 减小顶栏高度（当前偏厚）

#### 2.3 内容区域

- 背景使用 Canvas 色（暖灰），卡片使用纯白（`#ffffff`），形成清晰的 "卡片浮在灰底上" 的层次
- 卡片间距统一使用 `--space-5`（20px）
- 卡片圆角统一使用 `--radius-lg`（14px）
- 移除 `.card-lift` 的 hover 动画（管理面板不需要卡片漂浮效果）

### Phase 3: 各页面 UI 优化

#### 3.1 Dashboard（总览页）

**当前问题**: 信息太稀疏，Provider 列表平铺无层次。

**优化方案**:

```
┌──────────────────────────────────────────────────┐
│  [KPI 行 — 4 列卡片]                              │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐     │
│  │活跃Key  │ │冷却中   │ │失效Key  │ │Provider│     │
│  │  12    │ │   3    │ │   1    │ │   4    │     │
│  │ ↑ +2   │ │        │ │ ⚠     │ │        │     │
│  └────────┘ └────────┘ └────────┘ └────────┘     │
│                                                  │
│  [系统状态条 — 全宽]                               │
│  ● 系统正常  代理运行中  配置已同步  最后更新 14:32  │
│                                                  │
│  [Provider 卡片网格 — 2 列]                        │
│  ┌──────────────────┐ ┌──────────────────┐       │
│  │ ● OpenAI          │ │ ● Anthropic       │       │
│  │ → api.openai.com  │ │ → api.anthropic.. │       │
│  │                   │ │                   │       │
│  │ 🔑 8 活跃  2 冷却  │ │ 🔑 4 活跃  0 冷却 │       │
│  │ ⚡ 已激活          │ │    [激活]         │       │
│  │    [详情] [删除]   │ │    [详情] [删除]   │       │
│  └──────────────────┘ └──────────────────┘       │

```

- **KPI 卡片**: 大数字 + 趋势箭头 + 底部小字描述，背景纯白，左侧 4px 主色装饰条
- **Provider 卡片**: 从当前表格行改为独立卡片，每个 Provider 一张卡，包含健康状态、key 统计、快捷操作
- **空状态**: 无 Provider 时显示引导卡片（"添加你的第一个 Provider"），而非 Empty 组件

#### 3.2 Providers（提供商页）

**当前问题**: 773 行 God Component，表格行操作按钮拥挤，模态框嵌套深。

**优化方案**:

- **代码拆分**: 将 mutations 抽成 `useProviderMutations` 和 `useKeyMutations` 两个 hook，将模态框状态抽成 `useProviderModals` hook
- **表格优化**:
  - 行高增加到 56px
  - Key 数量用 **迷你条形图** 代替纯数字（绿色=活跃，橙色=冷却，红色=失效）
  - 操作按钮收纳到 **更多菜单** (···) 中，只保留最常用的一个外露
  - 模型列表改为可展开行（expand row），不用 tooltip
- **详情面板**: 从模态框改为 **右侧抽屉** (Drawer)，宽度 520px，不遮挡左侧列表
- **Key 管理表格**: 增加行内编辑能力（key 备注可直接点击编辑）

#### 3.3 Stats（调用统计页）

**当前问题**: KPI 卡片风格不统一，图表与表格缺少视觉节奏。

**优化方案**:

```

│  ┌──────────────────────────────────────────────┐│

│  │──────┼────────┼─────────┼──────┼─────┼──────││
```

- KPI 卡片统一为 **紧凑风格**: 上方标签 12px、中间数字 24px bold、下方趋势百分比
- 图表容器增加 **细微边框** 和 **圆角**，与 KPI 行形成视觉分组
- 表格增加 **斑马纹** (zebra striping) 提高可读性
- 延迟列用颜色编码：<1s 绿色, 1-3s 黄色, >3s 红色

#### 3.4 Settings（设置页）

**当前问题**: 表单平铺太长，分组感不够。

**优化方案**:

- 使用 **左侧标签导航 + 右侧表单** 的分栏布局（类似 macOS 系统偏好设置）
- 每个设置组用 **编号 + 标题 + 描述** 做区块头部
- "热生效" / "需重启" 标签改为 **彩色小药丸**，更醒目
- 底部固定 **保存栏** (sticky bottom bar)，显示 "已修改 X 项" + 保存/重置按钮
- 保存后结果用 **Toast** 而非顶部 Banner，减少视觉干扰

#### 3.5 Events（事件页）

**当前问题**: 工具栏筛选项排列太紧凑，详情抽屉信息层次不清。

**优化方案**:

- 工具栏改为 **两行布局**: 第一行搜索框全宽，第二行筛选条件 + 刷新控制
- 事件表格增加 **级别色条**（左侧 3px 装饰线：info=蓝, warn=橙, error=红）
- 详情抽屉使用 **分组手风琴** (Collapse) 展示 payload 的不同维度
- 修正 "每 8 秒自动刷新" 的文案为 "实时推送 (SSE)"

#### 3.6 About（关于页）

**当前问题**: 信息展示太朴素。

**优化方案**:

- 顶部 Hero 区域增加 **项目 Logo / 图标** + 版本号药丸
- 信息网格改为 **图标 + 标签 + 值** 三列布局
- 功能列表使用 **图标网格** 而非纯文字列表
- 备份操作按钮移到底部独立区块，增加操作确认

### Phase 4: 交互与动效提升

#### 4.1 表格交互

- **Hover 态**: 行背景从 `--mm-surface-active` 改为主色 4% 透明度 + 左侧 3px 主色指示条
- **选中态**: 复选框选中时整行背景加深
- **排序图标**: 替换 Ant Design 默认图标为更精致的箭头
- **分页器**: 居右对齐，增加 "共 X 条" 统计文本

#### 4.2 按钮体系

- 主按钮保持实心填充，圆角从 pill (24px) 改为 **rounded (10px)**——pill 形在管理面板中过于随意
- 次按钮使用 **outline 风格**（边框 + 透明底），而非当前的默认按钮样式
- 危险按钮统一为红色 outline
- 按钮组使用 `Space.Compact` 组合而非独立排列
- 移除 hover 时的 `translateY(-1px)` 上浮效果（管理面板不需要）

#### 4.3 过渡动画

- 页面切换: 改为 **淡入 + 微上滑** (120ms)，去掉当前的模糊 (blur) 效果
- 模态框: 使用 Ant Design 默认的 zoom 动画，保持统一
- 抽屉: 从右侧滑入，使用 `cubic-bezier(0.4, 0, 0.2, 1)` (Material 标准曲线)
- 数字变化: 保留 `anim-value-update`，但缩短到 200ms

#### 4.4 加载态

- 骨架屏覆盖所有页面（当前部分页面使用 `<Skeleton>`，部分用 `<Spin>`，不统一）
- 表格加载使用 **行内骨架** 而非全屏 Spin
- 按钮 loading 态禁用整个按钮并显示 spinner

### Phase 5: 代码质量改善（与 UI 同步进行）

#### 5.1 CSS 治理

- 清理 4 个暗色模式独有 token（`--mm-text-primary`, `--mm-surface-dark`, `--mm-border-dark`, `--mm-shadow-float`）
- 清理 3 个未使用的动画 token（`--ease-bounce`, `--ease-in-out`, `--duration-complex`）
- 合并重复 token（`--mm-bg` vs `--mm-body-bg`, `--mm-border-muted` vs `--mm-divider`）
- 消除 `#6366f1` 硬编码色值（`--kpi-accent` 的 fallback）
- 统一 monospace 字体栈（消除 `Fira Code` 的硬编码引用）
- 减少 `!important` 使用（shell 布局的 flex 覆盖改用更高优先级的选择器）
- 将 `providers.css` 中的 720px 媒体查询移到 `responsive.css` 统一管理

#### 5.2 组件拆分

- `providers-page.tsx` 拆分为:
  - `useProviderMutations` hook — 管理所有 provider 相关 mutation
  - `useKeyMutations` hook — 管理所有 key 相关 mutation
  - `useProviderModals` hook — 管理模态框开关状态
  - 页面组件本身只做布局编排和子组件组合

#### 5.3 内联样式清理

将以下组件的内联手写色值迁移到 CSS 变量:
- `provider-utils.tsx` 的 `stateColors` map
- `provider-modals.tsx` 的 `#3d8b50` / `#a83535`
- `save-summary-banner.tsx` 的 `#16a34a` / `#d97706`

## 实施优先级

| 批次 | 内容 | 预估工作量 | 视觉影响 |
|------|------|-----------|---------|
| **A** | Phase 1 (配色+字体+间距+阴影) | 中 | 极高 — 全局焕然一新 |
| **B** | Phase 2 (侧边栏+顶栏+布局) | 中 | 高 — 框架感改变 |
| **C** | Phase 3 各页面逐个优化 | 大 | 高 — 每页都有提升 |
| **D** | Phase 4 (交互+动效) | 小 | 中 — 细节打磨 |
| **E** | Phase 5 (代码质量) | 中 | 无 — 纯内部改善 |

建议执行顺序: **A → B → D → C (Dashboard → Providers → Stats → 其余) → E**

A+B 完成后整个界面的视觉基调和框架就会焕然一新，D 补充交互细节，C 逐页精修，E 收尾代码质量。

## 已确认的设计决策

| 决策项 | 选择 | 备注 |
|--------|------|------|
| **主色** | `#2563eb` 科技蓝 | 替换当前赤陶色 `#b85f45` |
| **背景基调** | 保留暖灰 | 底色维持暖色系，只拉大明度层次 |
